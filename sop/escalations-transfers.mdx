## SOP — L1 to L2 Transfer Protocol

### Before Transfer — L1 Steps

1. Collect AWB numbers, previous ticket IDs, and last resolution provided.
2. If no ticket exists — create a ticket with all details.
3. Inform seller you will check senior availability and ask for hold permission. Share hold time.
4. Mark **RED SOS** on the ticket.
5. Brief L2: AWB number, ticket number, VOC, total hold time.
6. Transfer call to L2.

### If L2 is Not Available

_Not applicable for Management, Consumer Forum, and Social Threat cases._

1. Apologize for the long hold.
2. Inform seller: All seniors are busy. I can arrange a callback within 30 minutes.
3. Fill the callback tracker.
4. Update ticket notes.
5. Mark **RED SOS** on the ticket.

### L2 Opening Script (to seller)

> Thanks for being on hold. Your call has been transferred to the supervisor desk. My name is _(name)_. How can I help you?

### L1 Briefing Script (to L2 — internal)

> Good morning / afternoon / evening. My name is _(name)_. I have Mr. / Ms. _(seller name)_ with me. They are facing _(issue)_. AWB: _(AWB number)_. Ticket ID: _(ID)_. Team gave an update on _(date)_. I informed them about the resolution, but they want to speak to a senior.

### L2 Takeover Script (to seller)

> Thanks for being on hold. Your call has been transferred to senior supervisor. I sincerely apologize for the time taken in transfer. Since your issue is _(describe)_, I would like to inform you that...

> **Important:** Social threats, Management escalations, and Consumer Forum cases require **live transfer**. If L2 is unavailable, TL / Mentor / QA / Trainer takes the call. No denial to seller.

## SOP — KAM Callback

If seller is requesting a callback from KAM or action is required from KAM:

| Field | Value |
|---|---|
| Properties Tag | Action by KAM |
| Ticket Status | Awaiting Customer Response |
| Group / Agent | Respective KAM Group and Name |

- Click **Reply to All** on the ticket.
- Remove seller's email ID from the reply.
- Add KAM's email ID.
- Include callback request details, Company ID, and registered number in notes.

**Callback TAT:** 24 working hours.

## SOP — Refund

### When to Give Refunds

- Seller is very upset, frustrated, or distressed in voice or communication.
- Seller asks for a refund immediately.
- Seller has contacted customer support **3 or more times** for the same issue.
- Cases involving ORM / Legal / Management threats.

### Refund Matrix

| Type | L1 Authority | L2 Authority |
|---|---|---|
| Freight Refund | Up to Rs. 500 | Up to Rs. 1,000 |

> L2 freight refund does not apply to heavy or large shipments over 10 kg.

### Cases Where Freight Refund Can Be Initiated on Seller Call

- EDD + 5 days and shipment has not been attempted.
- Without attempt RTO, whether LOPS reason is available or not.
- Shipment has BPR, less than 3 attempts, or zero attempts.

### Soft Loss — In Case of Dispute

- Full amount refunded at L2 if SLA is breached.
- Forward SLA: 7 days.
- RTO SLA: 15 days.

**Script if seller asks why a refund is being processed now:**

> We understand how frustrating this experience must have been. After reviewing the details shared by the courier and carefully assessing the situation, we sincerely regret the inconvenience. As a gesture of goodwill, we are processing a refund for you.

## SOP — OSLA Cases

If a seller approaches **outside of SLA (Out of SLA)**, tag the case as **OSLA** on FD (excluding international and buyer desk cases).

**Do NOT deny at any instance.**

**Sample OSLA scenarios:**
- TAT of 72 hours is breached for ACR response and seller approaches CC.
- Manifest was requested but seller approached after 72 hours of manifest request.
- TAT to raise weight or delivery dispute is breached.

## FCR / NFCR Tagging Guide

### General Queries (Order Creation, Rate Calculator, Cancellation Path)

| Field | Value |
|-------|-------|
| Channel | Customer Support Call Center |
| Agent Assign | Own Name |
| Status | Resolved |
| FCR | Yes |
| SR Issue Category | Other |
| Sub Category | General Query |

> Use this disposition only where DSAT chances are zero.

### Awaiting Customer Response (Recharge Failed, Damaged Shipment, Panel Functional Error)

**When documents need to be shared with any team:**

| Field | Value |
|-------|-------|
| Channel | Relevant Department |
| Agent Assign | Auto Assign |
| Status | Open |
| FCR | NFCR |

**When documents need to be checked:**

| Field | Value |
|-------|-------|
| Channel | Relevant Department |
| Agent Assign | Auto Assign |
| Status | Awaiting Customer Response |
| FCR | NFCR |

### FCR Cases (NDR Reattempt, RTO Status, Weight Discrepancy)

| Field | Value |
|-------|-------|
| Channel | Customer Support Call Center |
| Agent Assign | Own Name |
| Status | Resolved |
| FCR | Yes |

### NFCR Cases (Account Blocked, Delivery Delay, Tracking Update)

| Field | Value |
|-------|-------|
| Channel | Relevant Department |
| Agent Assign | Auto Assign |
| Status | Open |
| FCR | NFCR |

## Ticket Notes Format

All tickets must be created or assigned in Nugget using this format:

```
1. Seller VOC: [What the seller communicated]
2. Advisor's Response: [What action was taken]
3. Number of times seller called CC: [Count]
```
