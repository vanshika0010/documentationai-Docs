## Wallet Recharge

**Minimum recharge amount:** Rs. 500

**Path:** Seller Panel > Recharge Wallet

### Scenario 1 — Seller wants to know how to recharge

Guide the seller to the path above.

### Scenario 2 — Seller is unable to recharge

Request a screenshot within 48 hours of ticket creation. Assign to **Customer Support Tech Desk**. TAT: 24 hours.

## Recharge Error / Issue

### System Check

1. Copy link from SPARK and paste in a new tab.
2. Confirm when the seller recharged and the amount.
3. Check the payment gateway (Razorpay / Paytm) for transaction details.

### Razorpay Status Actions

| Status              | Action                                                                    |
| ------------------- | ------------------------------------------------------------------------- |
| Captured (over 1 hr)| Raise ticket to Tech Support to manually update the payment               |
| Captured (under 1 hr)| Ask seller to wait for the cron cycle to complete                        |
| Failed              | Inform seller: refund in 5–7 business days to source account              |
| Created / Pending   | Ask seller to wait 1 hour                                                 |
| Authorised          | Recharge has been done                                                    |

### Convenience Fee on Card Payments

A convenience fee is charged on Razorpay credit card transactions only.

- Fee is added automatically at Razorpay checkout after payment method selection.
- Not applicable on UPI, Net Banking, Debit Cards, Wallets, or Smart Collect.
- Reflected in Seller Panel > Passbook.
- Monthly VAS invoice generated for the fee.

**Example:** If seller recharges Rs. 1,00,000 via credit card, wallet gets credited with Rs. 97,640. A VAS entry of Rs. 2,360 is raised in real-time.

**Key dates:**
- Convenience fee started on 7th July 2025.
- Passbook entries automated from 14th August 2025.
- Additional UPI RuPay credit card check live from 28th August 2025.

### Scenario 2 — Seller has shared the transaction ID screenshot

> I am escalating your issue. Please wait 1–2 working days.

## Auto Wallet Transfer

Sellers can now initiate wallet refund requests directly from the panel.

**Path:** Seller Panel > Help and Support > Create Ticket > Finance > Transfer Wallet Amount into My Bank Account

The system automatically performs preliminary checks and calculates the eligible refund amount. A support ticket is generated only if validation fails.

---

## Wallet Balance Transfer Request

### Pre-Checks Before Raising Request

- No open shipments.
- Account status: Active.
- Bank / KYC details updated.
- COD unblocked.
- Wallet balance is positive (all linked accounts).

If all checks pass — raise to **Customer Support Finance Desk**. TAT: 1–2 working days.

If seller has coupon value in wallet:
> Sir / Ma'am, if you want to transfer your wallet amount to bank account, the coupon value will not be included in the transfer — only the wallet amount will be transferred.

### Nugget Ticket (NFCR)

| Field             | Value                         |
| ----------------- | ----------------------------- |
| Channel           | Customer Support Finance Desk |
| Agent Assign      | Priya / Manisha               |
| Status            | Open                          |
| Subject           | Wallet Balance Transfer       |
| FCR               | NFCR                          |
| SR Issue Category | Wallet Balance                |
| Sub Category      | Account to Account Wallet Transfer |

## Minimum Recharge Update

**Effective 14th July 2022:** Minimum recharge changed from Rs. 200 to **Rs. 500**.

**FAQ — Why was the denomination increased?**

> We have observed that most sellers do a minimum of 10–15 shipments a month and must recharge repeatedly. To avoid the hassle of recurring recharges, we have changed the recharge value from Rs. 200 to Rs. 500.

> Your amount is safe in our highly secured wallet, just like any other transactional wallet. You can use this amount for future shipments.
